# 防御模块说明（后端）

本文是防御塔和障碍物的说明文档。

没有设置怪物攻击防御塔的环节，即防御塔没有设置血量。

障碍物，想的逻辑是，玩家可以购买然后选择放在哪里以改变怪物的路线。
# 如果改为固定路径，删掉障碍物
# 如果不固定路径的话，其实还挺麻烦的，植物到底能不能吃，不能吃的话，那植物也变成障碍物的一种了，就要在地图层面规定哪些地方不能放植物。

## 1. 新构建了哪些类，分别在哪个文件

### 1.1 防御塔相关

- `DefenseTower`（抽象基类）
- `CoffeeTower`：小范围、极高单体伤害
- `AITower`：射程内 360° 扫射；豆包 → DeepSeek → GPT 可升级，等级越高扫射单发伤害与覆盖越强
- `LibraryTower`：半径内对怪物**减速**（不直接扣血）
- `ClassTower`：单次高伤、**长间隔**才打下一发
- `BilibiliTower`：仅沿**直线通道**攻击；放置后需用 `setFireDirection` 指定朝向；可花金币**改变射击方向**（90° 旋转）

定义位置：`DefenseTower.h`  
实现位置：`DefenseTower.cpp`

### 1.2 障碍物相关

- `Obstacle`（独立类，不继承防御塔）

定义位置：`Obstacle.h`  
实现位置：`Obstacle.cpp`

### 1.3 与敌人模块的联动

- `Enemy` 增加 `applySlowEffect` / `getEffectiveMoveSpeed`，供**图书馆塔**减速与 UI/调试显示移速。  
见：`Enemy.h` / `Enemy.cpp`

## 2. 每个类有什么数据成员和成员函数

### 2.1 `DefenseTower`（抽象基类）

#### 数据成员

- `towerName`：塔名称（用于日志/UI显示）。
- `cost`：购买该塔需要的金币。
- `position`：塔在地图上的中心坐标。
- `range`：攻击距离。对**圆型塔**为半径；**B 站塔**为沿射线方向的最远有效距离（见 B 站类说明）。
- `damage`：伤害或效果强度。图书馆塔在基类中记为 0（不通过 `takeDamage` 输出伤害）。
- `attackInterval`：效果冷却 / 攻击间隔（秒）。
- `cooldownTimer`：距上次成功发动效果的累计时间。
- `placed`：是否已放置到地图上。

#### 受保护工具

- `collectAliveInRange(...)`：收集**圆形范围内**所有仍存活的敌人（供 AI 扫射、图书馆减速等复用）。

#### 成员函数

- `place(const Vector2D& pos)`：放置塔，并预充冷却以便落子后很快出手。
- `update(float deltaTime, const std::vector<Enemy*>& enemies)`：默认流程为「冷却 → 选最近单一目标 → `attack`」。**AI / 图书馆 / B 站**子类**重写**了本函数以实现表内不同攻击方式。
- `attack(Enemy& target)`：默认对单目标 `takeDamage`；子类可覆盖；**图书馆**的 `attack` 为空（伤害由其他塔或机制处理）。
- `draw()`：纯虚函数，留给前端绘制。
- `canAttackNow()` / `isInRange()` / `pickTarget()`：辅助选敌与节奏控制（圆型、最近目标塔仍用默认 `update` 时走这套）。

### 2.2 `CoffeeTower`

- **设计对应**：范围小、瞬间伤害高；一次只打**最近**的一个目标（默认 `update` + 单目标 `attack`）。
- 主要函数：`CoffeeTower()`、`attack(Enemy&)`、`draw()`。

### 2.3 `LibraryTower`

- **设计对应**：一定半径内**全体减速**，不直接造成伤害。
- 数据成员：
  - `slowFactor`：移速倍率（与 `Enemy::applySlowEffect` 搭配，例如 0.45 表示约 45% 移速）。
  - `slowDurationSec`：每次生效时刷新的减速持续时间。
- 成员函数：
  - `update(...)`：**重写**，冷却到时对范围内每个敌人调用 `applySlowEffect`。
  - `attack(...)`：空实现（逻辑全部在 `update`）。
  - `draw()`：渲染预留。

### 2.4 `ClassTower`

- **设计对应**：单发伤害高，但 `attackInterval` 很长，两次出手间隔大。
- 主要函数：`ClassTower()`、`attack(Enemy&)`、`draw()`。

### 2.5 `BilibiliTower`

- **设计对应**：只在一条**带状直线**上命中敌人（射线方向 + 横向半宽 `kLaneHalfWidth`）；范围参数 `range` 表示沿射线从塔心向前的最大距离。
- 数据成员：`fireDirection`（归一化射击方向）。
- 成员函数：
  - `setFireDirection(const Vector2D& dir)`：放置后必须调用以瞄准（默认构造函数内为 +X，仍建议在关卡里显式设置）。
  - `getFireDirection()`：供前端画瞄准线。
  - `purchaseDirectionChange(int& playerGold, int goldCost)`：花金币将射击方向逆时针旋转 90°（可做「升级改朝向」的一种实现）。
  - `update(...)`：**重写**，在射线上选取**沿射线最近**的敌人攻击。
  - `attack(Enemy&)`：执行直线弹幕伤害。
  - `draw()`：渲染预留。

### 2.6 `AITower`（可升级）

- **设计对应**：每次开火对射程内所有敌人各结算一次伤害（**360° 扫射**）；升级提高单次扫射伤害、射程与攻速。
- 数据成员：`level`、`maxLevel`、`upgradeCosts`、`levelNames`（Doubao / DeepSeek / GPT）。
- 成员函数：
  - `canUpgrade` / `upgrade` / `nextUpgradeCost` / `getLevel` / `getMaxLevel`：升级与经济结算。
  - `update(...)`：**重写**，一次扫射对 `collectAliveInRange` 内全部敌人 `takeDamage(damage)`。
  - `attack(Enemy&)`：保留单目标接口（如需手动点名），平时由扫射 `update` 驱动。
  - `draw()`：渲染预留。

### 2.7 `Obstacle`（障碍物）

#### 数据成员

- `cost`、`position`、`width`、`height`、`blocksPath`、`placed`。

#### 成员函数

- `place`、`getBlockingRect`、`isBlocking`、`isPlaced`、各类 getter。

## 3. 给主循环同学：常用接口

### 3.1 放置与购买

- `tower.getCost()`、`tower.place(pos)`。
- **BilibiliTower**：`place` 之后务必调用 `setFireDirection(...)`（或由 UI 传入选方向向量）。
- `obstacle.getCost()`、`obstacle.place(pos)`。

### 3.2 每帧驱动

- 敌人：`enemy.update(deltaTime)`
- 防御塔：`tower.update(deltaTime, enemyPointers)`
- **AI 塔升级**：`canUpgrade` / `upgrade`。
- **B 站改变朝向（升级）**：`purchaseDirectionChange(playerGold, cost)`。

### 3.3 图书馆减速可在 UI 上显示的数值

- `enemy.getEffectiveMoveSpeed()`：当前理论移动速度（用于验证减速是否生效）。

## 4. 给地图/寻路同学：常用接口

- 障碍物：`getBlockingRect()`，放置合法性仍由地图模块校验（与此前一致）。

## 5. 给前端同学：常用接口

### 5.1 绘制相关

- 所有塔仍实现 `draw()`（当前空壳）。
- **BilibiliTower**：用 `getFireDirection()` + `getPosition()` + `getRange()` 画射线与扇形/矩形通道（横向宽度可取类内 `kLaneHalfWidth` 的两倍作为可视化参考）。

### 5.2 展示面板可读取的数据

- 通用：`getName()`、`getCost()`、`getPosition()`、`getRange()`、`getDamage()`、`getAttackInterval()`、`isPlaced()`。
- AI 塔：`getLevel()`、`getMaxLevel()`、`nextUpgradeCost()`。
- B 站塔：`getFireDirection()`；升级改向按钮可调用 `purchaseDirectionChange`。
- 障碍物：同前。

### 5.3 当前实现状态说明

- **AI**：控制台会打印扫射命中人数；伤害对范围内**每个**敌人各结算一次。
- **图书馆**：仅减速；可与 `getEffectiveMoveSpeed()` 联调表现。
- **咖啡 / 上课**：经典单体最近目标模式；数值强调「短射程高爆发」与「长间隔重击」。
- **B 站**：直线命中；需对接瞄准 UI。
